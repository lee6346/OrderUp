bin/: 
    - directory for defining any startup scripts
    bin/www:
    - startful file that creates/runs the server instane with port configurations and injects the app.js modules 
    bin/test, bin/stop, bin/restart, etc:
    - different startup files you can place in the bin folder to for running test server, stopping server, etc 
    
    package.json-> scripts{}: change the start, debug, test, etc to these files 
    Procfile-> web: npm start, test: npm test, etc -> for heroku apps 

creates the server instance, port configs,  and passes the app.js module into it 
    - separates middleware from running the app to maintain independently
    - one exception is that 
app.js: contains all middleware and mvc routes
